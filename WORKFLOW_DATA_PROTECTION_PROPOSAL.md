# Workflow Data Protection Proposal

**Date**: 2025-11-10
**Issue**: @profcryptotalks KOL lost during xai-trackkols → master merge
**Root Cause**: Workflow-generated data overwritten by worktree merge

---

## Problem Statement

### What Happened

1. **Nov 10, 00:30 UTC**: KOL Discovery workflow ran on `master`, adding @profcryptotalks
   - master: 14 KOLs
2. **Later**: xai-trackkols branch merged into master (commit 0fea15a)
   - xai-trackkols had branched off before @profcryptotalks was discovered
   - Merge overwrote `kols-data.json` with older version from worktree
   - **Result: @profcryptotalks DELETED**
3. **Now**: master has 13 KOLs (data loss confirmed)

### Why It Happened

The `merge-worktree.mjs` script lacks protection for workflow-generated data files:
- ✅ Protects worktree-specific files (`.env.worktree`, etc.)
- ✅ Protects main branch config files (`test/.env`, etc.)
- ❌ **NO protection for workflow data files** (`kols-data.json`, `kol-replies.json`, etc.)

---

## Solution: 3-Layer Protection System

### Layer 1: Pre-Merge Data Sync (CRITICAL)

**Add automatic master sync before merging worktree back**

Location: `scripts/worktree/merge-worktree.mjs`, add after line 255

```javascript
// NEW: Fetch latest master and check for workflow data updates
console.log(colorize('\n🔄 Syncing with latest master branch...', 'yellow'));

try {
    // Fetch latest master from origin
    execCommand('git fetch origin master:master', { ignoreError: true });

    // Check if master has new commits since worktree branched
    const masterNewCommits = execCommand(`git rev-list --count ${branchName}..master`);

    if (parseInt(masterNewCommits) > 0) {
        console.log(colorize(`⚠️  Master has ${masterNewCommits} new commits since '${branchName}' branched`, 'yellow'));

        // Check if workflow data files were modified
        const workflowDataChanged = execCommand(
            `git diff --name-only ${branchName}..master | grep -E "scripts/kols/data/(kols-data|kol-replies|processed-posts)\\.json"`,
            { ignoreError: true }
        );

        if (workflowDataChanged) {
            console.log(colorize('\n⚠️  WORKFLOW DATA CONFLICT DETECTED!', 'red'));
            console.log(colorize('   Master has new workflow-generated data:', 'yellow'));
            workflowDataChanged.split('\n').forEach(file => {
                console.log(colorize(`   - ${file}`, 'yellow'));
            });

            if (!updateFlag) {
                console.log(colorize('\n❌ Merging would overwrite workflow data!', 'red'));
                console.log(colorize('\n💡 Options:', 'cyan'));
                console.log('  1. Merge master INTO worktree first (RECOMMENDED):');
                console.log(`     cd .trees/${branchName}`);
                console.log(`     git merge master`);
                console.log(`     cd ../..`);
                console.log(`     npm run worktree:merge ${branchName}`);
                console.log('  2. Use --update flag to auto-merge master into worktree:');
                console.log(`     npm run worktree:merge ${branchName} -- --update`);
                process.exit(1);
            }

            // If --update flag is set, merge master into worktree first
            console.log(colorize('\n🔄 Auto-merging master into worktree (--update flag)...', 'cyan'));
            const worktreePath = join(rootDir, '.trees', branchName);

            try {
                execCommand(`git -C "${worktreePath}" merge master -m "Merge latest workflow data from master"`);
                console.log(colorize('✅ Worktree updated with latest master data', 'green'));
            } catch (mergeError) {
                console.error(colorize('\n❌ Auto-merge failed - conflicts detected', 'red'));
                console.error(colorize('   Resolve conflicts manually:', 'yellow'));
                console.error(`   cd ${worktreePath}`);
                console.error('   git status');
                console.error('   # resolve conflicts');
                console.error('   git add <files>');
                console.error('   git merge --continue');
                process.exit(1);
            }
        } else {
            console.log(colorize('✅ No workflow data conflicts', 'green'));
        }
    }
} catch (error) {
    console.log(colorize('⚠️  Could not sync with master - continuing with merge', 'yellow'));
}
```

**Benefits**:
- ✅ Detects workflow data changes before merge
- ✅ Forces developer to sync worktree with latest master first
- ✅ Prevents data loss automatically
- ✅ Works with `--update` flag for automatic sync

### Layer 2: Workflow Data File Protection

**Add new protection category for workflow-generated files**

Location: `scripts/worktree/merge-worktree.mjs`, add after line 48

```javascript
// NEW: Files generated by GitHub Actions workflows that need special merge handling
const WORKFLOW_GENERATED_FILES = [
    'scripts/kols/data/kols-data.json',
    'scripts/kols/data/kol-replies.json',
    'scripts/kols/data/processed-posts.json',
    'scripts/kols/data/engaging-posts.json',
    'scripts/kols/config/x-crawler-accounts.json',
    'scripts/data/discovered-paths.json',
    '_posts/**/*.md',  // Articles generated by workflows
    '_posts/articles/**/*.md'
];

function isWorkflowGeneratedFile(file) {
    return WORKFLOW_GENERATED_FILES.some(pattern => {
        // Handle glob patterns
        if (pattern.includes('*')) {
            const regex = new RegExp('^' + pattern.replace(/\*/g, '.*').replace(/\//g, '\\/') + '$');
            return regex.test(file);
        }
        // Exact match
        return file === pattern;
    });
}
```

### Layer 3: Data Loss Detection

**Add pre-commit validation to detect data loss**

Location: `scripts/worktree/merge-worktree.mjs`, add before line 498 (before commit)

```javascript
// NEW: Validate workflow data before committing
console.log(colorize('\n🔍 Validating workflow data integrity...', 'yellow'));

const dataLossDetected = [];

// Check KOL count
try {
    const masterKols = JSON.parse(
        execCommand('git show master:scripts/kols/data/kols-data.json')
    ).kols.length;

    const mergedKols = JSON.parse(
        readFileSync(join(rootDir, 'scripts/kols/data/kols-data.json'), 'utf8')
    ).kols.length;

    if (mergedKols < masterKols) {
        dataLossDetected.push({
            file: 'kols-data.json',
            type: 'KOLs',
            before: masterKols,
            after: mergedKols,
            lost: masterKols - mergedKols
        });
    } else if (mergedKols > masterKols) {
        console.log(colorize(`   ✅ KOLs: ${masterKols} → ${mergedKols} (+${mergedKols - masterKols})`, 'green'));
    } else {
        console.log(colorize(`   ✅ KOLs: ${masterKols} (unchanged)`, 'green'));
    }
} catch (error) {
    // File might not exist or be unparseable - skip validation
}

// Check reply count
try {
    const masterReplies = JSON.parse(
        execCommand('git show master:scripts/kols/data/kol-replies.json')
    ).replies.length;

    const mergedReplies = JSON.parse(
        readFileSync(join(rootDir, 'scripts/kols/data/kol-replies.json'), 'utf8')
    ).replies.length;

    if (mergedReplies < masterReplies) {
        dataLossDetected.push({
            file: 'kol-replies.json',
            type: 'Replies',
            before: masterReplies,
            after: mergedReplies,
            lost: masterReplies - mergedReplies
        });
    } else if (mergedReplies > masterReplies) {
        console.log(colorize(`   ✅ Replies: ${masterReplies} → ${mergedReplies} (+${mergedReplies - masterReplies})`, 'green'));
    } else {
        console.log(colorize(`   ✅ Replies: ${masterReplies} (unchanged)`, 'green'));
    }
} catch (error) {
    // Skip validation if file doesn't exist
}

// If data loss detected, ABORT
if (dataLossDetected.length > 0) {
    console.log(colorize('\n❌ DATA LOSS DETECTED!', 'red'));
    console.log(colorize('   Merging would delete workflow-generated data:', 'red'));

    dataLossDetected.forEach(loss => {
        console.log(colorize(`\n   File: ${loss.file}`, 'yellow'));
        console.log(colorize(`   ${loss.type}: ${loss.before} → ${loss.after} (${loss.lost} lost)`, 'red'));
    });

    console.log(colorize('\n💡 This usually means:', 'cyan'));
    console.log('   - Workflows added data to master while you were working');
    console.log('   - Your branch needs to merge master first');
    console.log('   - You have conflicting local changes that removed data');

    console.log(colorize('\n🔧 To fix:', 'cyan'));
    console.log(`   cd .trees/${branchName}`);
    console.log('   git merge master');
    console.log('   # Resolve any conflicts, keeping both sets of data');
    console.log('   git push');
    console.log('   cd ../..');
    console.log(`   npm run worktree:merge ${branchName}`);

    console.log(colorize('\n⚠️  Aborting merge to prevent data loss', 'yellow'));
    execCommand('git merge --abort', { ignoreError: true });
    process.exit(1);
}

console.log(colorize('✅ Data integrity verified', 'green'));
```
