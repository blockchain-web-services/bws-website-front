# Plan: Enhanced Content Structural Diversity for KOL Replies

## Current Problem Analysis

### Issues Identified:
1. **Uniform sentence length**: All paragraphs have same structure (1 sentence para 1, 1-2 sentences para 2)
2. **Repetitive opening pattern**: Second paragraph ALWAYS starts with "$BWS [Product Name]"
3. **Format monotony**: No variation in formatting (emojis, lists, structure)

### Examples of Current Monotony:

**Reply 1**:
```
100x requires separating signal from noise - especially in microcaps.

$BWS X Bot tracks KOL performance with official X API metrics...

@BWSCommunity #gems #microcap #crypto
```

**Reply 2**:
```
Supply shocks are the perfect setup for capital rotation...

$BWS Fan Game Cube enables sports clubs to monetize...

@BWSCommunity #Bitcoin #microcap #NFT
```

**Pattern**:
- Opening: 1 sentence
- Middle: Always starts "$BWS [Product]..."
- Format: Plain text, no variation

## Proposed Solution: Structural Templates System

### Core Concept
Create 5-8 **structural templates** that define:
- Paragraph count (2-4 paragraphs)
- Sentence length per paragraph (1-3 sentences)
- Product mention position (beginning, middle, end)
- Format type (plain, emoji, bullet list, question-answer, etc.)

### Template Categories

#### 1. **Classic Structure** (Current - 30% usage)
```
[1 sentence context]

$BWS [Product]: [2 sentences features/benefits]

@BWSCommunity [hashtags] [URL]
```

#### 2. **Feature List Structure** (15% usage)
```
[1-2 sentences context]

[Product name] delivers:
• [Feature 1]
• [Feature 2]
• [Feature 3]

Real utility at microcap valuation. $BWS

@BWSCommunity [hashtags] [URL]
```

#### 3. **Problem-Solution Structure** (15% usage)
```
[Problem statement - 1-2 sentences]

Here's the solution: [Product] from $BWS [specific capability].

[1 sentence additional detail or benefit]

@BWSCommunity [hashtags] [URL]
```

#### 4. **Question-Led Structure** (10% usage)
```
[Engaging question related to tweet?]

$BWS [Product] answers this: [specific features]. [Additional sentence].

[Optional third paragraph with angle]

@BWSCommunity [hashtags] [URL]
```

#### 5. **Stat/Fact-Driven Structure** (10% usage)
```
[Agree/Context - 1 sentence]

Consider this: $BWS [Product] [specific metric/capability]. [Product] [technical detail or unique angle].

@BWSCommunity [hashtags] [URL]
```

#### 6. **Emoji-Enhanced Structure** (10% usage)
```
[Context emoji] [1 sentence context]

💡 $BWS [Product]: [feature statement]
🔧 [Technical detail]
📈 [Benefit/angle]

@BWSCommunity [hashtags] [URL]
```

#### 7. **Delayed $BWS Mention** (10% usage)
```
[Context - 1-2 sentences, NO $BWS yet]

[Product name] [specific features]. This is what $BWS ships - [additional detail].

@BWSCommunity [hashtags] [URL]
```

### Phase 2: Template-Aware Prompt Generation

**File**: `scripts/crawling/utils/claude-client.js`

**Changes needed**:
1. Load `reply-templates.json`
2. Select random template based on weights
3. Check last N replies to avoid template repetition
4. Generate template-specific prompt instructions

**New function**:
```javascript
function selectReplyTemplate(recentReplies) {
  // Load templates
  const templates = loadReplyTemplates();

  // Get templates used in last 5 replies
  const recentTemplateIds = extractTemplateIds(recentReplies.slice(0, 5));

  // Filter out recently used templates
  const availableTemplates = templates.filter(t =>
    !recentTemplateIds.includes(t.id)
  );

  // Weighted random selection
  return weightedRandomSelect(availableTemplates);
}
```

### Phase 3: Enhanced Prompt Construction

**Modified prompt sections**:

#### Add Template Instructions Section
```
**STRUCTURAL TEMPLATE FOR THIS REPLY**: ${template.name}

Structure Requirements:
- Paragraphs: ${template.structure.paragraphs}
- Opening: ${template.structure.opening.sentences} sentence(s), ${template.structure.opening.pattern}
- Middle: ${template.structure.middle.format || 'paragraph'} format
${template.formatting.bulletPoints ? '- Use bullet points (•) to list features' : ''}
${template.formatting.emojis ? '- Include 1-2 relevant emojis' : '- No emojis'}
- $BWS mention: ${template.structure.middle.startsWithCashtag ? 'Start middle section' : 'Include naturally in middle/end'}
```

#### Add Variation Examples Per Template
```
**${template.name} EXAMPLES**:
${template.examples.map((ex, i) => `
Example ${i+1}:
"${ex.text}"
`).join('\n')}
```

#### Update Diversity Requirements
```
**STRUCTURAL DIVERSITY REQUIREMENTS**:
1. Template for THIS reply: ${template.name}
2. Recent templates used: ${recentTemplateIds.join(', ')} - DO NOT use these
3. Sentence length: VARY between short (5-10 words) and medium (15-25 words)
4. $BWS placement: ${cashtagPlacementRule(template)}
5. Paragraph count: ${template.structure.paragraphs} paragraphs
${template.formatting.bulletPoints ? '6. Format features as bullet list with •' : ''}
${template.formatting.emojis ? '6. Include emojis: Use 1-2 contextual emojis (e.g., 💡📊🔧⚡)' : ''}
```

## Risk Mitigation

### Potential Issues:

1. **Character limit violations** (280 chars)
   - Solution: Template instructions must emphasize brevity
   - Each template includes max character guidance

2. **Template weights not working**
   - Solution: Log template selection to verify distribution
   - Adjust weights based on real usage patterns

3. **Claude not following template structure**
   - Solution: Provide clear examples per template
   - Add structural validation in response parsing

4. **Over-complicated replies**
   - Solution: Keep templates professional and clear
   - Avoid templates that sacrifice clarity for variety
