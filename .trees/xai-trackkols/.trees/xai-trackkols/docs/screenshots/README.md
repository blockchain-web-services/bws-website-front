# Visual Snapshots Directory

This directory contains automated visual snapshots of all pages across different viewports for mobile CSS review and responsive design QA.

## Directory Structure

```
screenshots/
├── main/                    # Main pages (/, /about, /contact-us, etc.)
│   ├── index/
│   │   ├── desktop.png
│   │   ├── iphone-se.png
│   │   ├── iphone-12-pro.png
│   │   └── pixel-5.png
│   ├── about/
│   └── ...
├── industry-content/        # Industry pages (/industry-content/*)
│   ├── content-creation/
│   ├── esg/
│   └── ...
├── marketplace/             # Marketplace pages (/marketplace/*)
│   ├── blockchain-badges/
│   ├── esg-credits/
│   └── ...
└── articles/                # Article pages (/articles/*)
    ├── article-x-bot/
    └── ...
```

## Viewport Sizes

Each page has 4 screenshots at different viewport sizes:

- `desktop.png` - 1920×1080px (standard desktop)
- `iphone-se.png` - 375×667px (small mobile)
- `iphone-12-pro.png` - 390×844px (standard mobile)
- `pixel-5.png` - 393×851px (Android mobile)

## Automatic Updates

Snapshots are automatically regenerated by GitHub Actions when:

1. **On push to main** - When source files change in `src/` or `public/`
2. **Weekly schedule** - Every Sunday at 2 AM UTC
3. **Manual trigger** - Via workflow_dispatch in GitHub Actions

See `.github/workflows/update-snapshots.yml` for details.

## Manual Generation

### Prerequisites
```bash
# Build the site first
npm run build

# Ensure Playwright browsers are installed
cd tests
npx playwright install chromium
```

### Generate All Snapshots
```bash
# From project root
npm run snapshot:update

# Or from tests directory
cd tests
npm run test:snapshot
```

### Generate Specific Pages
```bash
cd tests

# Single page
npm run test:snapshot -- --grep "index"

# Single viewport
npm run test:snapshot -- --grep "desktop"

# Specific page and viewport
npm run test:snapshot -- --grep "index.*desktop"
```

## Troubleshooting

### WSL/Local GPU Issues

If you encounter GPU crashes when running locally (especially on WSL), this is a known Playwright limitation in virtualized environments.

**Solution implemented:**
- Homepage mobile snapshots use height-limited capture (15000px max) to prevent GPU crashes
- This affects the homepage only, which is 19000-22000px tall on mobile viewports
- All other pages use full-page screenshots without limitations
- The height limit captures 67-79% of homepage content, sufficient for CSS review

**Additional workarounds if needed:**
- Test one viewport at a time with `--workers=1` flag
- Use `--grep "desktop"` to test desktop only (less resource-intensive)

### Missing Screenshots

If screenshots aren't appearing after running the test:

1. Ensure the site is built: `npm run build`
2. Check the preview server is working: `cd tests && npm run preview:sirv`
3. Visit http://localhost:4321 to verify the site loads
4. Re-run: `npm run snapshot:update`

### Screenshot Quality

Snapshots are full-page screenshots at actual device viewports. File sizes typically range from:

- Desktop (1920×1080): 2-5MB per page
- Mobile (375-393px width): 500KB-2MB per page

Total directory size for all 96 snapshots: approximately 100-200MB.

## Git LFS Consideration

If the repository size becomes an issue due to screenshots, consider using Git LFS:

```bash
git lfs install
git lfs track "docs/screenshots/**/*.png"
```

Currently, screenshots are committed directly to the repository for easy viewing in GitHub.
